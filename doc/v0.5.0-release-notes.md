## Highlights

### Core Flow Coverage

- D4 logic pack verification is green on latest macOS + Apple Silicon:
  - `93 passed / 0 failed` (includes key-action menu availability coverage)
- Coverage includes:
  - key generation/import/export
  - trust/signing core logic

- Backup/Restore entry is now visible from key action menu in primary flow.

- Key management footer and app title are now explicit localized keys.

### Security Hardening

- Sanitized key-generation batch fields to reduce payload-injection risk.
- Reduced sensitive identity leakage in logs by removing per-key identity output.

## Known Issues

1. Signed/notarized packaging is deferred in current environment.
   - Impact: this GitHub release ships an unsigned `Debug` DMG artifact.
   - Follow-up: configure `MOAIY_DEVELOPMENT_TEAM_ID` in `Moaiy/Config/Moaiy.Signing.xcconfig` to enable signed/notarized release track.
2. `RB-002` sandbox-only macro expansion failure during restricted `xcodebuild` runs.
   - Impact: environment-specific test instability under sandbox constraints.
   - Product risk: low; non-sandboxed runs pass.
   - Workaround: run release validation outside restricted sandbox.
3. Compatibility matrix still needs external environment closure:
   - macOS 12.x smoke
   - real Intel machine smoke (currently deferred because no Intel environment is available)
   - Fallback evidence for pre-release: `x86_64` destination smoke `93/93`, `failed=0`
## Upgrade Notes

- No data migration is required for existing keyring data.
- Backup validation is still recommended before production usage:
  - create backup
  - perform restore dry run on a disposable keyring
