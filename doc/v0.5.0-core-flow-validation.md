# v0.5.0 Core Flow Validation List

## Purpose

Define a single source of truth for release-blocking validation flows in v0.5.0.

## Release-Blocking Flows

### 1. Key Generation

- Create key with valid name/email/key type
- Verify key appears in list after refresh
- Verify key details render correctly

### 2. Key Import/Export

- Import ASCII-armored public key
- Import ASCII-armored secret key (with passphrase handling)
- Export public key to file
- Export secret key to file (with passphrase prompt)

### 3. Text Encryption/Decryption

- Encrypt text for selected recipient
- Decrypt generated ciphertext successfully
- Validate user-facing error when recipient/passphrase is invalid

### 4. File Encryption/Decryption

- Encrypt a plain text file
- Decrypt back to original file content
- Validate overwrite/path conflict handling

### 5. Backup/Restore

- Create backup package with public keys only
- Create backup package including secret keys
- Restore from generated backup package
- Validate partial-failure reporting when some secret-key exports fail

### 6. Key Trust and Signing

- Update trust level for a key
- Sign a target key and verify signature state updates

## Exit Criteria

Release can proceed only when:

- All six core flow groups pass on at least one primary environment
- No `P0`/`P1` issues remain open for these flows
- All failures (if any) are documented in Known Issues with severity and workaround
