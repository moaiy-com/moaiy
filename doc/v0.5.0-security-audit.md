# v0.5.0 Lightweight Security Audit

## Scope

Audit date: 2026-04-04  
Scope: D3 release checklist items for command execution, sensitive data handling, temp storage hygiene, and sandbox permissions.

## Findings

### SAF-001 (Fixed): Batch-parameter injection surface in key generation payload

- Severity: `P2`
- Area: `GPGService.buildKeyGenerationParams`
- Risk:
  - User-provided `name` / `email` values were inserted into GPG batch payload lines directly.
  - Newline/control characters could alter batch payload semantics.
- Fix:
  - Added `sanitizeBatchField(_:)` and applied it to `Name-Real` / `Name-Email` fields.

### SAF-002 (Fixed): Sensitive key identity details in logs

- Severity: `P2`
- Area: `KeyManagementViewModel`
- Risk:
  - Logs included key fingerprint and user identity (`name <email>`), which can leak sensitive metadata in diagnostics.
- Fix:
  - Removed per-key identity logging.
  - Replaced export logs with non-identifying messages.
  - Replaced `print` in trust-check error path with structured logger output.

## Verification Summary

- Command execution path uses `Process` with argument arrays (no shell interpolation).
- No passphrase persistence to `UserDefaults` or filesystem-backed app settings was found.
- Temporary operation directories use `SecureTempStorage` and are cleaned with `defer` + stale cleanup.
- App entitlements remain minimal for current feature scope:
  - `com.apple.security.app-sandbox`
  - `com.apple.security.files.user-selected.read-write`
  - `com.apple.security.network.client`

## Remaining Notes

- `RB-002` (sandbox-only macro expansion failure in restricted test runs) remains environment-specific and non-release-blocking for product behavior.
