## Closed Blockers

### RB-004: Secret-key import via UI failed in manual regression (`MR-03`)

- Severity: `P1` (release blocking)
- Status: Closed
- Date discovered: 2026-04-04
- Date resolved: 2026-04-04
- Evidence:
  - Initial fail report: `dist/manual-ui/v0.5.0-manual-user-manual-20260404-150210/manual-report.md`
  - Symptom: `Import Key` with `sec.asc` returned user-visible error `key import failed`.
- Root cause:
  - Secret-key import path depended on agent/pinentry availability; packaged runtime could fail with agent-unavailable behavior.
- Resolution:
  - Updated import commands to loopback-safe mode in `GPGService`:
    - `--batch --yes --pinentry-mode loopback --passphrase '' --import --status-fd 1`
  - Applied to both file import and armor-data import paths.
  - Kept parser fix for full `IMPORT_RES` status-field offsets.
- Verification:
  - Automated: targeted suites passed
    - `xcodebuild test -project /Users/codingchef/Taugast/moaiy/Moaiy/Moaiy.xcodeproj -scheme MoaiyTests -destination 'platform=macOS' -only-testing:MoaiyTests/GPGServiceTests -only-testing:MoaiyTests/KeyImportExportTests`
  - Manual: user confirmed `MR-03` secret-key import success on a newly repackaged DMG (reported on 2026-04-04).
