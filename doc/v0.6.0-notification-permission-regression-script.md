# v0.6.0 Notification Permission Regression Script

This runbook validates the reminder-permission flow introduced in v0.6.0,
with focus on the "permission denied" branch and prompt consistency.

## Scope

- Reminder toggle rollback when notification permission is denied.
- Prompt visibility and copy consistency after denial.
- Authorized path behavior (toggle stays enabled and reminders can be scheduled).

## Automated Integration Tests

Run the permission flow integration suite:

```bash
xcodebuild test \
  -project /Users/codingchef/Taugast/moaiy/Moaiy/Moaiy.xcodeproj \
  -scheme MoaiyTests \
  -destination 'platform=macOS' \
  -only-testing:MoaiyTests/ExpirationReminderServicePermissionFlowTests
```

Expected result:

- `** TEST SUCCEEDED **`
- Three cases pass:
  - denied status rolls back toggle
  - notDetermined + user deny rolls back toggle
  - authorized status keeps toggle enabled and schedules reminders

## Manual Regression Bundle

Prepare a manual run bundle (report template + optional permission reset):

```bash
./scripts/v060_prepare_notification_permission_manual.sh --label local
```

Optional parameters:

```bash
./scripts/v060_prepare_notification_permission_manual.sh \
  --label qa-runner-a \
  --bundle-id com.moaiy.app \
  --app-path /Applications/Moaiy.app
```

The script generates:

- `dist/manual-ui/v0.6.0-notification-permission-<label>-<timestamp>/manual-report-template.md`
- `dist/manual-ui/.../README.md`

## Manual Test Cases

- NP-01: Toggle reminder on, choose "Don't Allow" in system permission prompt.
- NP-02: Confirm reminder toggle rolls back to disabled state.
- NP-03: Confirm in-app guidance prompt appears for denied permission.
- NP-04: Confirm prompt style (title, body, button style) matches global prompt host style.
- NP-05: Confirm no reminder request is effectively scheduled under denied permission.
- NP-06: Reset permission, repeat flow with "Allow", confirm toggle remains enabled.
