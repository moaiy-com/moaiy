# Moaiy 产品设计方案

## 品牌定位

### 品牌名称：Moaiy
**源自摩艾石像 (Moai)，象征神秘感、安全性、长久性**

**品牌口号**: *"像摩艾一样守护您的秘密"*

#### 品牌核心价值
- 🗿 **神秘感**: 守护用户的数字秘密，让敏感信息永远只属于用户自己
- 🔐 **安全性**: 提供军事级别的加密保护，让数据固若金汤
- ⏳ **长久性**: 设计理念是长期可靠，一次加密，永久保护

## 产品定位

### 核心理念
**技术应该隐形，用户只关注"我要做什么"**

### 产品目标
1. **零门槛**: 双击即用，无需任何技术背景
2. **开箱即用**: 完全自包含，无外部依赖
3. **智能化**: 自动处理复杂配置和错误
4. **安全性**: 使用最强加密算法，不妥协安全性

## 目标用户画像

### 主要用户群体
- **加密货币用户**: 需要安全存储助记词、私钥
- **隐私保护者**: 希望加密个人文件和通信
- **普通办公用户**: 需要加密敏感文档

### 用户痛点分析
| 痛点 | 现状 | 期望 |
|------|------|------|
| 安装复杂 | 需要安装GPG、配置环境 | 双击即用 |
| 操作复杂 | 需要记忆命令和参数 | 图形界面、拖拽操作 |
| 术语难懂 | 公钥、私钥、签名等概念 | 日常语言、直观展示 |
| 错误难解 | 晦涩的错误代码 | 友好提示、自动修复 |
| 密钥管理 | 手动管理文件和目录 | 可视化列表、自动备份 |

## 用户体验设计原则

### 1. 极简主义
- 界面清爽，只显示必要信息
- 隐藏高级选项，按需展开
- 减少用户决策点

### 2. 向导式流程
- 复杂操作分解为简单步骤
- 每步都有清晰说明
- 提供合理的默认值

### 3. 智能化
- 自动检测用户意图
- 智能推荐最佳方案
- 自动处理常见问题

### 4. 容错性
- 防误操作设计
- 友好的错误提示
- 自动修复机制

### 5. 一致性
- 统一的操作逻辑
- 一致的视觉风格
- 符合macOS设计规范

## 主要用户流程

### 1. 首次使用流程
```
启动应用 
  → 欢迎界面
  → 自动创建默认密钥
  → 快速教程
  → 进入主界面

总耗时: < 30秒
用户操作: 点击2-3次
```

### 2. 文本加密流程
```
复制要加密的文本
  → 打开应用（或菜单栏工具）
  → 点击"快速加密"
  → 文本自动粘贴并加密
  → 结果自动复制到剪贴板

总耗时: < 5秒
用户操作: 拖拽或粘贴
```

### 3. 文件加密流程
```
拖拽文件到应用窗口
  → 自动识别文件类型
  → 显示加密选项（使用默认值）
  → 点击"加密"
  → 自动保存到指定位置

总耗时: < 10秒
用户操作: 拖拽 + 点击
```

### 4. 硬件密钥设置流程（Pro版）
```
插入硬件密钥
  → 自动检测并提示
  → 启动快速设置向导
  → 设置PIN码（或使用默认）
  → 生成密钥
  → 完成设置

总耗时: < 2分钟
用户操作: 点击3-4次
```

## 开源策略

### 开源部分
```
gpg-manager/
├── Core/              # 核心功能（开源）
│   ├── GPGService/   # GPG命令行封装
│   ├── Models/       # 数据模型
│   └── Utils/        # 工具类
├── UI/                # 基础 UI（开源）
│   ├── Views/        # 界面组件
│   └── ViewModels/   # 业务逻辑
└── Services/          # 基础服务（开源）
    ├── FileService/  # 文件操作
    └── BackupService/ # 备份服务
```

### 闭源部分（Pro版）
```
Pro/
├── HardwareKey/       # 硬件密钥管理
├── CloudSync/         # 云端同步
└── AdvancedFeatures/  # 高级功能
```

### 许可证选择

#### 推荐方案： GPLv3
**优势**:
- 要求衍生作品也开源
- 防止闭源商业化
- 符合开源精神

**劣势**:
- 限制商业使用
- 可能影响企业用户

#### 替代方案： MIT
**优势**:
- 更宽松
- 允许商业使用
- 更容易被接受

**劣势**:
- 可能被闭源商业化
- 失去控制权

## 风险评估

### 技术风险
- **沙盒限制**: 可能影响功能完整性 → 需要创新解决方案
- **硬件兼容性**: 不同品牌密钥可能有差异 → 广泛测试
- **性能问题**: 大文件加密可能耗时 → 异步处理 + 进度显示

### 商业风险
- **市场竞争**: 现有免费工具竞争 → 差异化定位
- **用户付费意愿**: 免费版功能已足够 → Pro功能必须有价值
- **App Store审核**: 可能遇到审核问题 → 提前准备合规材料

### 法律风险
- **加密软件出口**: 需要遵守美国出口法规 → 正确填写合规问卷
- **开源许可证**: 需要遵守依赖库许可证 → 仔细审查
